# Reference Thumbnail Playbook

「センスの良いサムネイル」を安定して再現するための社内用メモ。競合（ColdFusion / AI Explained / ゆっくり解説系）を観察したときに共通していたポイントを整理し、レイアウト選択やQAに落とし込んでいます。

## 1. レイアウトパターン

| Pattern | 使いどころ | ルール |
| --- | --- | --- |
| L-Split (左テキスト / 右キャラ) | 対話系・解説系の基本型 | ・テキスト領域は幅の55%<br>・キャッチは2行まで<br>・右下に男女キャラ＋吹き出し |
| Diagonal Stripe | 緊急ニュース / 勢いを出したい時 | ・左上↘右下のストライプで動きを付ける<br>・フォーカス色は3色以内<br>・数字を大きく配置 |
| Badge Stack | How-to / Tips系 | ・左上にカテゴリバッジ、右上に数字バッジ<br>・中央にサンプル画像 or アイコン |

## 2. Typography

- メインキャッチは **8〜12文字 / 2行以内**。行間はフォントサイズの 0.8 倍。
- キーワードのジャンプ率は 1.6〜2.0。`重要語 > 補足語 > サブコピー` の順でサイズを固定。
- ゴシック体＋太字（NotoSansCJK W6推奨）。すべて大文字にしない。句読点は極力排除。

## 3. Color & Texture

- ベース / アクセント / ハイライト + 白黒の **最大4色**。アクセントを背景と文字で重複させない。
- 背景は暗め→文字を明るく。奥行き用に **1) グラデーション 2) パターン 3) 光の輪** を重ねる。
- カテゴリに合わせたテーマ例:
  - 経済: ネイビー + ゴールド
  - テック: シアン + パープル
  - カルチャー: コーラル + アイボリー

## 4. キャラクター配置

- 「男性:説明」「女性:共感」など、音声の役割と一致させる。
- 片側だけを極端に大きくしない（対角で 10〜15px ずらす）。
- キャラの下に白いラウンドバブル + 影で地面感を出す。

## 5. QA チェックシート

1. **読みやすさ**: 角度やパターンで文字が潰れていないか。
2. **余白**: キャッチ上下に 40px 以上の余白があるか。
3. **装飾の意味**: ただの線や図形ではなく、視線誘導や情報整理に機能しているか。
4. **キャラ一貫性**: サムネの性別と TTS の声が一致しているか。
5. **カテゴリの明示**: バッジやサブコピーでジャンルが1秒で伝わるか。

## 6. 参考収集ワークフロー

1. Pinterest / Behance / Xで「経済 ニュース サムネ」「Yukkuri 解説」で検索。
2. 良例をPNG保存 → `docs/reference_thumbnails/examples/<category>/yyyy-mm-dd_<source>_<desc>.png` に格納（命名ルールは `examples/README.md` を参照）。
3. 本READMEに以下情報を追記:
   - どのレイアウトに分類されるか
   - 使われている色パレット
   - 文字数とジャンプ率

> ⚠️ ネット接続できない環境では、紙のムードボードや既存YouTubeのスクショをオフライン保管して参照してください。
